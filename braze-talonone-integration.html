1. Overview: Summary & Business Outcomes

The integration allows for a seamless flow of data between the customer journey (Braze) and the incentive logic (Talon.One). Instead of static, generic discounts, promotions become dynamic, secure, and deeply integrated into the user experience.

Key Business Outcomes

Reduced Promotion Leakage

Real-time validation ensures codes are used only by eligible customers.

Increased Conversion Rates

Deliver high-relevance incentives via the right channel at the moment of intent.

Enhanced Loyalty Engagement

Use real-time ledger updates to trigger "Milestone" or "Tier Upgraded" messages.

Operational Agility

Marketers launch campaigns via the Rule Builder without requiring dedicated engineering sprints.

2. Feature Connectivity & Data Exchange

The integration primarily utilizes Braze's Connected Content for pulling data and Webhooks/Notifications for pushing data.

Braze Feature Talon.One Feature Data Exchanged Direction
Connected Content API Endpoints Coupon codes, Loyalty balances, Referral codes Braze ← Talon.One
Canvas Webhooks / Notifications Triggering "Reward Earned" or "Tier Upgraded" events Talon.One → Braze
Email / Push / SMS Personalization (Liquid) Displaying unique codes or point balances in-message Talon.One → Braze
Audiences Customer Profiles Syncing Braze segments to Talon.One for eligibility Braze → Talon.One

3. How to Setup: Guidelines & Data Flows

To achieve a successful integration, both platforms must use a shared Integration ID (typically the Braze external_id).

Scenario A: Customer Profile Registration & Sync

Keep Talon.One attributes in sync with Braze user profiles to power rule-based targeting.

Data Flow: Profile Sync
1
User Action
Customer updates profile (e.g., changes city, upgrades tier) or signs up
↓
2
Braze
Attribute change triggers a Webhook Campaign in Braze
↓
3
Braze → Talon.One
Braze sends PUT request to Talon.One Customer Profile API with updated attributes
↓
4
Talon.One
Updates customer profile with new attributes (Target_Segment, Last_City, etc.)
↓
5
Talon.One
Profile is now available for rule-based targeting in campaigns
Setup Steps:
  • Create a Braze Webhook Campaign triggered by an attribute change or sign-up event
  • Configure endpoint: PUT https://[your_deployment].talon.one/v1/customer_profiles/{{${user_id}}}
  • Set Content-Type to application/json
  • Add your Talon.One API Key in the headers
Example Payload:
{ "attributes": { "Target_Segment": "High_Value", "Last_City": "{{${most_recent_location}}}" } }

Scenario B: Transactions & Real-time Rewards

Trigger loyalty points or rewards immediately after a purchase.

Data Flow: Real-time Rewards
1
User Action
Customer completes a purchase transaction
↓
2
E-commerce Platform
Sends transaction data to Talon.One via Session Update (Close)
↓
3
Talon.One
Evaluates rules and calculates rewards (e.g., 50 loyalty points earned)
↓
4
Talon.One → Braze
Sends webhook notification to Braze /users/track endpoint with points_earned event
↓
5
Braze
Receives event and triggers "Points Earned" Canvas journey
↓
6
Braze
Sends personalized notification (Email/Push/SMS) to customer with reward details
Setup Steps:
  • In Talon.One, create a Notification webhook triggered by "Session Update" (Close)
  • Configure webhook to send to Braze /users/track endpoint
  • Use Braze Data Transformations to map Talon.One's payload to valid Braze event format
  • Create a Canvas in Braze that triggers on the points_earned event
Talon.One Notification Payload Example:
{ "events": [ { "external_id": "profile.IntegrationID", "name": "points_earned", "properties": { "amount": 50 } } ] }

Scenario C: Audience Sync

Synchronize Braze Segments to Talon.One Audiences for exclusive promotional eligibility.

Data Flow: Audience Sync
1
Braze
Segment is created/updated (e.g., "Churn_Risk" segment identified based on user behavior)
↓
2
Talon.One
Create corresponding Audience in Talon.One (e.g., "Churn_Risk" Audience with unique ID)
↓
3
Braze
Create a Canvas with Webhook tile at segment entrance
↓
4
User Action
Customer enters the "Churn_Risk" segment in Braze
↓
5
Braze → Talon.One
Canvas webhook fires, adding user to Talon.One Audience via API
↓
6
Talon.One
User is now member of "Churn_Risk" Audience and eligible for targeted campaigns
↓
7
Talon.One
Rules can now use condition: "Customer is in Audience: Churn_Risk" for exclusive offers
Setup Steps:
  • In Talon.One, create an Audience (e.g., "Churn_Risk") and note the Audience ID
  • In Braze, create a Canvas with a Webhook tile at the segment entrance
  • Configure webhook to call Talon.One's Audience API to add the user
  • Use the condition in Talon.One rules: "Customer is in Audience: Churn_Risk"

4. Infrastructure: Performance & Reliability

Sub-100ms Latency

Talon.One is engineered for high-velocity environments. This ensures that when Braze makes a Connected Content call to fetch a coupon during a message send, the response is near-instant, preventing timeouts and ensuring a smooth user experience.

Resilient Webhook Retry Logic

  • Retry Policy: Talon.One attempts to resend a failed webhook up to 10 times or until a 2xx success response is received.
  • Exponential Backoff: There is an initial 5-minute gap between attempts. This gap grows with each attempt, giving the receiving system (Braze) time to recover without being overwhelmed.
